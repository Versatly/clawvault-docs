## Introduction

ClawVault is a **structured memory system** for AI agents, designed to persist decisions, context, and knowledge across sessions. It composes into any agent framework through a CLI-first approach.

## Installation

A minimum version of Node.js 18 is required.

```bash
npm install -g clawvault
```

Optionally install [qmd](https://github.com/tobi/qmd) for semantic search:

```bash
npm install -g github:tobi/qmd
```

## Setup

<Steps>
  <Step>
    ### Initialize your vault

    ```bash
    clawvault init ~/memory
    ```

    This creates the vault directory with `.clawvault.json` config and category folders.
  </Step>
  <Step>
    ### Start a session

    ```bash
    clawvault wake
    ```

    Wake checks for context death, recovers any pending checkpoints, and loads recent context.
  </Step>
  <Step>
    ### Store memories

    ```bash
    clawvault remember decision "Use PostgreSQL" \
      --content "Need concurrent writes for multi-agent setup"

    clawvault capture "Look into Redis for caching layer"
    ```

    Use `remember` for structured memories with a type, `capture` for quick notes.
  </Step>
  <Step>
    ### Search your vault

    ```bash
    # Keyword search
    clawvault search "database"

    # Semantic search (requires qmd)
    clawvault vsearch "what did we decide about storage"
    ```
  </Step>
  <Step>
    ### End your session

    ```bash
    clawvault sleep "Completed database selection" \
      --next "Set up connection pooling" \
      --decisions "PostgreSQL chosen over SQLite"
    ```

    Sleep creates a handoff with context for the next session.
  </Step>
</Steps>

## For OpenClaw Users

ClawVault integrates with [OpenClaw](https://openclaw.ai) through the hook system:

```bash
clawhub install clawvault
```

This provides automatic checkpoint before `/new`, context death detection on startup, and session start context injection. See the [Hook Setup](/integrations/openclaw/hook-setup) guide.
