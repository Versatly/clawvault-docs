# template

Use `template` to list available templates, create files from templates, and add your own custom templates.

## Usage

```bash
clawvault template list [options]
clawvault template create <name> [options]
clawvault template add <file> --name <name> [options]
```

## `template list`

List built-in and custom templates.

```bash
clawvault template list
```

| Flag | Description | Default |
|------|-------------|---------|
| `-v, --vault <path>` | Vault path | Nearest vault |

## `template create`

Create a new document from a template.

```bash
# Create from a built-in template
clawvault template create decision --title "Use Postgres over MongoDB"

# Create a daily note
clawvault template create daily-note --title "2026-02-15"
```

| Flag | Description | Default |
|------|-------------|---------|
| `-t, --title <title>` | Document title | Auto-generated |
| `-v, --vault <path>` | Vault path | Nearest vault |

## `template add`

Add a custom template from an existing file.

```bash
clawvault template add ~/my-templates/standup.md --name standup
```

## Built-in Templates

Current built-ins:

- `checkpoint`
- `daily-note`
- `decision`
- `handoff`
- `lesson`
- `person`
- `project`

## Details

Custom templates are stored in your vault's `templates/` directory. Once added, they become available through `template create <name>`.

<Callout type="info">
Keep templates minimal and schema-focused. Overly complex templates slow down capture and reduce adoption.
</Callout>
